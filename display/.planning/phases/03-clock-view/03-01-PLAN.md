```
while True:
    now = time.ticks_ms()
    if _pending_dir != 0:
        _current_idx = (_current_idx + _pending_dir) % 3
        _pending_dir = 0
        VIEWS[_current_idx].render(oled)
        _draw_page_dots(oled, _current_idx)
        oled.show()
    if weather_view.should_refresh(now):
        weather_view.refresh(oled)
        VIEWS[_current_idx].render(oled)
        _draw_page_dots(oled, _current_idx)
        oled.show()
    if _current_idx == 1 and clock_view.should_tick(now):
        clock_view.render(oled)
        _draw_page_dots(oled, _current_idx)
        oled.show()
    if clock_view.should_sync(now):
        clock_view.sync(oled)
    time.sleep_ms(_POLL_MS)
```

Order rationale — `should_tick` fires BEFORE `should_sync` in the loop body so a freshly-synced time renders on the next iteration's minute check, not on the same tick as the sync (avoids visible "sync flicker" on the tick where sync happens to fall on a minute boundary).

`from main import TZ_OFFSET` note — MicroPython allows this circular-looking import because `main.py` is executed as `__main__`, and by the time `clock_view.render` is first CALLED (from the poll loop below `if __name__ == "__main__":`), `main.py` has already fully executed its top-level assignments. `clock_view.py` is imported at line 4 of `main.py`, BEFORE `TZ_OFFSET` is assigned at line 12, so a naive `from main import TZ_OFFSET` at the top of `clock_view.py` would fail with `ImportError` — the attribute doesn't exist yet at that moment. Two viable resolutions:

1. **Lazy import inside `render`** (D-38 baseline): `from main import TZ_OFFSET` on the first line of `render(oled)` and again on the first line of `should_tick(now_ms)`. Each call re-imports the same reference — cheap, guaranteed available at call time.
2. **Move `TZ_OFFSET` to a dedicated module** (e.g. `config.py`) that both `main.py` and `clock_view.py` import at load time. Cleaner but introduces a new file.

This plan uses **option 1** (lazy import inside `render` and `should_tick`) for minimum churn — no new file, no config-module design decision unmade later. Executor may switch to option 2 if a compelling reason surfaces during implementation, but D-38's canonical refs assume option 1.

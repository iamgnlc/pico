<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → device filesystem | secrets.py is copied to the Pico via mpremote; on-device it is world-readable to any process — acceptable because the device is single-tenant. |
| repo → git remote | Anything committed becomes durable in refs. The whole point of this plan is to keep secrets.py OUT of this boundary. |
| host filesystem → editor buffers | If secrets.py is opened in an editor with cloud sync (e.g. GitHub Codespaces, Copilot indexing), the value can egress. Not remediable at the plan level; user acknowledged. |

From display/sh1107.py — OLED inherits from framebuf.FrameBuffer. Signature of interest:
```
oled.ellipse(cx, cy, rx, ry, color, fill)
  cx, cy: center coordinates in framebuf pixel space
  rx, ry: radii in pixels
  color:  0 or 1
  fill:   True = filled disk, False = hollow ring (WE WANT False, per D-06)
```

Canonical usage examples in display/icons.py:
- Line 22: `fb.ellipse(cx, cy, 5, 5, 1, True)`      — sun body (filled)
- Line 27: `fb.ellipse(cx, cy, 10, 10, 1, True)`    — moon body (filled)
- Line 28: `fb.ellipse(cx + 5, cy - 2, 9, 9, 0, True)` — moon crescent cutout (filled with color 0)
- Line 32: `fb.ellipse(x + 10, y + 18, 6, 5, 1, True)` — cloud left puff (filled)
- Line 33: `fb.ellipse(x + 16, y + 14, 8, 7, 1, True)` — cloud center puff
- Line 34: `fb.ellipse(x + 22, y + 18, 6, 5, 1, True)` — cloud right puff

Our degree ring uses the same idiom but with `fill=False` (hollow) and small radius (rx=ry=2).

Existing display/main.py:_render() structure (BEFORE this plan, POST-Plan-01):
- Line ~22 def _render(oled):
- Line ~23 oled.fill(0)
- Line ~24 ip = wifi.connect(secrets.WIFI_SSID, secrets.WIFI_PASSWORD)   # touched by Plan 01
- Line ~25-26 if not ip: _center_text(oled, "no wifi", WIDTH // 2, HEIGHT // 2)
- Line ~27-30 else: temp, code, is_day = weather.current(); if temp is None: "no data"
- Line ~31-33 icons.draw(oled, 16, 16, code, is_day); _center_text(oled, "{:.0f}C".format(temp), 88, HEIGHT // 2, scale=2)
- Line ~34 oled.show()

`_center_text` (main.py:16-19) computes: `w = 8 * len(s) * scale; h = 8 * scale; text_render.text(oled, s, x_center - w // 2, y_center - h // 2, scale)`. So the text's LEFT edge is at `x_center - w // 2` and the RIGHT edge is at `x_center + w // 2`. At scale=2 with a 2-digit temperature ("19"), w = 8 * 2 * 2 = 32, so the right edge of the digits is at x_center + 16 = 88 + 16 = 104. The baseline (bottom of text) is at y_center + h // 2 = HEIGHT // 2 + 8 = 40. The top of text is at y_center - h // 2 = 32 - 8 = 24.

Ring placement (D-06, D-08 — "top-right of the digits at scale 2", "small horizontal gap"):
- cx = right edge of last digit + gap + rx  →  approximately 104 + 3 + 2 = 109 (a 3px gap between the digit's right edge and the leftmost pixel of the ring)
- cy = top of text + rx                     →  approximately 24 + 2 = 26 (ring sits high, superscript position)
- rx = ry = 2 (D-06)
- color = 1, fill = False

Handle 1-digit and 3-digit temperatures the same way by using `w = 8 * len(s) * 2` at the call site rather than hardcoding 104. Compute the string once, use its length for w, then reuse the string for the text draw.

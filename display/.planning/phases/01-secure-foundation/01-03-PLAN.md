## Phase Goal

**As a** developer about to make this repo public, **I want to** expunge the historically-committed WiFi credentials from every reachable commit and update the remote to match, **so that** no attacker can `git log -S` my old password out of the repo's history after publication.

<objective>
Rewrite the outer /Users/gnlc/Code/pico repo's git history using `git filter-repo` to replace every occurrence of the two credential strings (`REDACTED_SSID` SSID and `REDACTED_PASSWORD` password) with `REDACTED_SSID` and `REDACTED_PASSWORD` respectively (per D-09). After a blocking human checkpoint confirms the router-side rotation has occurred (D-11), force-push all branches and tags to origin (D-10) and drop the reflog + filter-repo backup refs so the old strings are truly unreachable.

Purpose: SEC-03. This is Slice B from the planning brief. It runs LAST in the phase (Wave 3) because D-10 and D-11 require the code-level extraction (Plan 01) and the degree symbol fix (Plan 02) to be committed cleanly first, so the rewrite operates over a known-good tree with everything in place.

Output: No file changes — the artifact of this plan is a rewritten commit graph. Verified via `git log --all -S 'string' -- '*'` queries.

**This plan is NOT autonomous.** It contains a blocking human checkpoint to confirm that the router-side credential rotation has already been completed. Force-push must not run against creds that are still live on the network — that would leave the old creds exposed at all currently-reachable clones for the window between the push and rotation.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local git object store → remote refs | Force-push transitions destructive local rewrites into the remote. Any client with a stale clone is invalidated. |
| host filesystem → /tmp | The redact.txt replacements file itself contains the sensitive strings on the LEFT side of `==>`. If left on disk, it leaks. Task 2 mandates deletion. |
| filter-repo backup refs → reachability | filter-repo stores original refs under refs/original/ by default. Task 2 step 5 explicitly expires and gc's these so the old strings become unreachable. |
| live WiFi network → attacker | If a copy of history escaped BEFORE the router-side rotation, the credentials in that copy are still live on the network. D-11 blocks this by requiring rotation FIRST. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | The old strings persist in a filter-repo backup ref or the reflog | mitigate | Task 2 step 5 runs `git reflog expire --expire=now --all` and `git gc --prune=now --aggressive` explicitly. Task 2 acceptance verifies via `git log --all -p | grep -c` that the strings are absent across all reachable objects. |
| T-01-03-02 | Information Disclosure | /tmp/pico-redact.txt lingers on the host and leaks the strings via file backup or Time Machine | mitigate | Task 2 explicitly deletes the file after successful rewrite; Task 2 acceptance verifies file absence. |
| T-01-03-03 | Denial of Service | Force-push clobbers unpushed work on another clone | mitigate | Task 3 checkpoint reconfirms solo-remote and gives the operator one last chance to reconcile before pushing. |

| T-01-03-05 | Repudiation | The rewrite produces a subtly different tree than the operator expected (e.g. changes a binary that happened to contain the substring) | mitigate | Task 2 step 4 requires manual inspection; `git status` clean after rewrite means no unexpected working-tree drift; the REDACTED_SSID / REDACTED_PASSWORD counter check confirms the replacement worked. |

D-11 sequence discipline ensures the rotation lands first. |

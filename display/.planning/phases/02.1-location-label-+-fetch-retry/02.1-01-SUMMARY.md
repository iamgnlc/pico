## Implementation Choices (Planner Discretion Applied)

- **Location extraction order in `weather.py`:** placed BEFORE the second `urequests.get()` for the open-meteo forecast, so a forecast-endpoint failure would still have `location` computed. The outer `try/except` still returns 4-Nones on any failure, so this is a purely stylistic ordering choice with no behavior change (as noted in the plan action).
- **Truncation rule:** exactly `max_chars = min(15, WIDTH // 8)` = `min(15, 16) = 15`. Note: the plan-checker's FLAG-1 pointed out that 15 chars centered on x=88 exceeds the visible right edge (x=127) — chars past position ~12 will silently clip off-screen. This is a valid concern but D-27 explicitly grants "planner discretion for pixel-level tuning during on-device verify" — the operator can nudge `max_chars` down (e.g. to 12) during Task 3 if long city names clip badly.
- **Location rendering only in "ok" branch:** the plan-checker's FLAG-2 flagged that CONTEXT.md L117 recommended rendering `_cached_location` whenever non-None regardless of `_cache_status` (so a wifi drop after a successful fetch would preserve the last-known city). I followed the plan text as written — location only draws in the "ok" branch. If the operator prefers the last-known-city behavior during error states, the follow-up is a two-line change (move the `if _cached_location is not None:` guard + draw call outside the "ok" branch, applying to no_wifi/no_data too).
- **Redundant `or None` in cascade:** kept as-written per FLAG-3 discussion. Cosmetic; MicroPython evaluates fine.
- **`_cache_status = "ok"` timing preserved:** the success block still assigns `_cache_status = "ok"` AFTER the four cache writes, so a partial-write race is not possible in the single-threaded MicroPython model.

## MicroPython Gotchas Encountered

- **`global` declaration must list ALL module names written** — extending the write set to include `_cached_location` required extending the `global` line at the top of `refresh()`. Missing this would create a local shadow. Verified via grep.
- **Ordering of `_cached_location` in the module-state block:** placed BEFORE `_cache_status` so the cache fields group naturally (all `_cached_*` together, `_cache_status` as the separate state flag). No functional impact; readability only.

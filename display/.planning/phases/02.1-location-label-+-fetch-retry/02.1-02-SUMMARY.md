# Plan 02.1-02 Summary — Fetch Retry Cadence (WEATHER-09)

## What Shipped

`weather_view.py` now selects the refresh interval by inline-reading `_cache_status`:

```
def should_refresh(now_ms):
    interval = _REFRESH_MS if _cache_status == "ok" else _RETRY_MS
    return time.ticks_diff(now_ms, _last_refresh_ms) >= interval
```

Two module-level constants:
- `_REFRESH_MS = 600_000` — unchanged (WEATHER-03, from Phase 2)
- `_RETRY_MS = 60_000` — new (WEATHER-09)

Everything else is byte-identical.

## Requirements Delivered

- **WEATHER-09**: When a weather fetch fails, the device retries every 60 seconds until the first successful fetch, then reverts to the 600-second default cadence.

Code paths satisfy WEATHER-09; end-of-phase on-device human-verify still pending per the deferred batch.

## Decisions Honored

- **D-31**: Predicate inline-reads `_cache_status`; no boolean flag, no mutable interval state.
- **D-32**: Cold-boot failure treated symmetrically — first failure at boot triggers 60s retry mode without special-case logic (verified structurally; on-device confirmation deferred).
- **D-33**: `_last_refresh_ms = time.ticks_ms()` stamped at the START of `refresh(oled)` — unchanged. Retry cadence is measured from attempt start, not from attempt end.

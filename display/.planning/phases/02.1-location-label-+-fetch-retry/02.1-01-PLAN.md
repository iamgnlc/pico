## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ip-api.com → weather.py | External HTTP JSON response is parsed into a dict; untrusted string fields (`city`, `regionName`, `countryCode`) are read from it and stored in module state. |
| weather_view module state → OLED framebuffer | `_cached_location` is written to the SPI-backed 1024-byte framebuffer via `text_render.text()`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02.1-01 | Tampering | ip-api response `city`/`regionName`/`countryCode` fields | mitigate | Truncate via `[:max_chars]` where `max_chars = min(15, WIDTH // 8) = 15`; framebuf-native text rendering ignores non-ASCII glyphs (draws whitespace); no `eval`/format-injection surface — string is only passed to `text_render.text()` for pixel rendering. |
| T-02.1-02 | Information Disclosure | Location string on the OLED | accept | Location is city-level (not GPS coord); already implicitly exposed by any observer with visual access to the device. Rationale: no PII beyond what an observer already knows about the device's rough location. |
| T-02.1-03 | Denial of Service | Very long ip-api string field | mitigate | Hard cap `[:max_chars]` slice bounds the render cost regardless of upstream string length; no unbounded loop. |
| T-02.1-04 | Tampering | Broad `except Exception:` in `weather.current()` | accept | Preserved deliberately per D-29; any exception (network, JSON parse, KeyError) collapses to the 4-None sentinel, which the caller treats as `_cache_status = "no_data"`. Distinguishing error taxonomies is deferred (see CONCERNS.md). |
| T-02.1-SC | Tampering | npm/pip/cargo installs | mitigate | N/A — this phase adds no new packages. All modules used (`urequests`, `time`) are MicroPython stdlib; no installer touchpoint. |

Zero HIGH severity threats. `security_asvs_level: 1`, `security_block_on: high`: no gating checkpoints required for this plan beyond the standard visual verify.

</threat_model>

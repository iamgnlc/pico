## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| SUMMARY quotes redacted strings | When writing a SUMMARY that documents a credential/secret redaction, the summary itself can re-leak the values by quoting them in "here's the replacements file we used" or shell command examples. This happened in the first draft of `01-03-SUMMARY.md` and the verifier caught it after the SUMMARY was committed and pushed. | advisory | Future SUMMARY authors: describe redactions structurally ("both credential strings"), do not enumerate the strings. If a table or code block is helpful, use `[old ssid]` / `[old password]` placeholders. Verifier should run a `git log --all -S` sweep as part of `passed` verdict for any phase that touches secrets. |

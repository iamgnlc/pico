## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| ip-api default fields omit `offset` and `query` | ip-api's `/json/` endpoint returns only status/country/countryCode/region/regionName/city/zip/lat/lon/timezone/isp/org/as/query by default. Requesting `offset` (tz) and `query` (WAN IP) requires an explicit `?fields=` query param. Once caused a clock-stuck-at-`--:--` regression (Plan 03-02, fix c23bb5a). | advisory | Any future ip-api integration in `bootstrap.py` (or a successor) must specify `?fields=<comma-separated>` in the URL. Current `bootstrap.fetch()` uses `?fields=lat,lon,offset,query` — do not drop it. |
| Two-step refactor that leaves the tree temporarily broken between steps | When step A removes a symbol that step B replaces (e.g. `weather_view.refresh` removed in T-1, `_refresh_all` added in T-2), committing A alone leaves `main.py` non-parsing. | advisory | Either commit A+B as a single atomic unit (preferred — see 260719-f0b's c78b12c) OR call it out explicitly in the plan's task ordering notes and note that intermediate ast.parse verify will fail (see 260719-e5g's T-1+T-2 split). |
| Views importing sibling views | Prior shape: `views/weather_view.py` imported `clock_view` + `system_view` to push tz_offset/wan_ip into their setters after fetching. Coupling smell — views should be leaves. | advisory | Fixed in quick 260719-f0b. `main.py` is now the composition root: its `_refresh_all(oled)` calls `bootstrap.fetch()` and fans out to `weather_view.set_data`, `clock_view.set_tz_offset`, `system_view.set_wan_ip`. New views must follow this template — see `.planning/codebase/ARCHITECTURE.md` for the current shape. |
| Stale docs after a structural refactor | `.planning/codebase/*.md` referenced `weather.py` / `wifi.py` after the 2026-07-18 bootstrap consolidation, and referenced flat view paths after the 2026-07-19 views/ package move. | resolved-2026-07-19 | Fixed by running `/gsd:map-codebase --refresh` at the end of this session (commit 816ddcd). If a future refactor reshapes module boundaries, run `/gsd:map-codebase` before the codebase-context docs are consumed by any planner/discuss workflow. |
| gsd-sdk binary not installed on this host | Workflows nominally require `gsd-sdk` in PATH. Local host uses the legacy `~/.claude/get-shit-done/bin/gsd-tools.cjs` fallback via `node`. All workflows still function; some newer commands (`gsd-sdk query worktree.cleanup-wave`) are unavailable. | advisory | Install with `npm install -g get-shit-done-cc` if a future workflow errors out at Step 2. Not blocking for `/gsd:quick`, `/gsd:map-codebase`, `/gsd:resume-work`, or `/gsd:pause-work`. |

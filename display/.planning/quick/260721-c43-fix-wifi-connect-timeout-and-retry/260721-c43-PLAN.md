<objective>
Fix intermittent "no wifi" on the display caused by (a) Pico W CYW43 associations that tail past the current 20 s timeout, and (b) the absence of any radio reset when the first association attempt lands in a stuck CYW43 state. Both are additive changes to `bootstrap._wifi_connect` only.

Purpose: The operator reports that the Pico W is close to the router and every other device on the same network connects reliably, yet the display sometimes lands on "no wifi". The two failure modes (timeout too short for the CYW43 tail; no reset on stuck-state failures) are diagnosed in the orchestrator's turn and both are addressed by this fix. The happy path (already-connected mid-session, or fast cold association) is not slowed down.

Output: A modified `bootstrap.py` with an updated `_wifi_connect` implementation. Every other file in the repo is byte-identical.
</objective>

  <action>
Modify `_wifi_connect` in `bootstrap.py` (currently lines 6-15). Only this function changes; the rest of the module (imports, `fetch()`, everything at line 17 and below) MUST remain byte-identical.

Required behavioral additions (both must land):

1. Bump the default initial timeout from 20 s to 30 s. Keep the parameter name `timeout` and its default position (third positional). This catches the tail of typical CYW43 cold associations (15-25 s median → 90th-percentile around 30 s) without slowing the happy path.

2. On failure of the initial polling window, perform ONE full radio-reset retry before returning None:
   - `wlan.disconnect()` — clear any queued/stuck CYW43 attempt
   - `wlan.active(False)` — tear the interface down
   - `time.sleep(1)` — let the CYW43 firmware settle
   - `wlan.active(True)` — bring the interface back up fresh
   - `wlan.connect(ssid, password)` — re-issue with the same credentials
   - Poll again with a shorter secondary window (10 s), same 1-second cadence as the initial loop
   - Then return the IP if connected, else None

Preserved behaviors (do NOT regress any of these):
- If `wlan.isconnected()` is already True on entry, the function returns the IP immediately with no reset, no additional sleeps, no radio churn. This is the happy path for the 600 s scheduler-tick refresh cadence — critical.
- The signature stays `_wifi_connect(ssid, password, timeout=30)`. Callers in `fetch()` and anywhere else are unchanged.
- Return is a string IP on success or `None` on failure. `bootstrap.fetch()` at line 33 uses `if not ip:` so both `None` and empty-string would trigger the fallback, but keep the string-or-None contract.
- `wlan = network.WLAN(network.STA_IF)` — the CYW43 station interface. Do not swap to AP mode.
- Total wall-clock worst case on the failure path: `30 + 1 + 10 = 41 s` (initial + settle + retry). This is within the ≤ 45 s bound.

Numeric values (recommended defaults — use these unless you have a strong reason within the sanctioned bands below):
- Initial timeout: **30 s** (band: 25-45 s; do NOT exceed 45 s — cold boot users watch the "connecting..." screen through this)
- Settle delay between `active(False)` and `active(True)`: **1 s** (band: ≥ 500 ms)
- Retry timeout after reset: **10 s** (band: 5-15 s)

Structural notes:
- The retry poll loop can be an inline second `for _ in range(retry_timeout):` — no need to factor a helper; this function stays short.
- MicroPython idiom: no type hints, no docstring. Keep the existing style. Feel free to add a brief inline comment explaining the reset-and-retry (one to three lines) since this is a non-obvious CYW43 recovery.
- Do NOT log, print, or introduce any new module import. `network` and `time` are already imported at the top of bootstrap.py.
- Do NOT touch `bootstrap.fetch()`, imports, or any code after line 15's original slot.

Referential shape of the resulting function (semantics only — the executor writes the actual code following existing style):
- Get wlan handle, `wlan.active(True)`.
- If `wlan.isconnected()`: return the IP immediately (fast path unchanged).
- Otherwise: `wlan.connect(...)`, poll up to `timeout` seconds (default 30). If connected, return the IP.
- Otherwise: `wlan.disconnect()`, `wlan.active(False)`, `time.sleep(1)`, `wlan.active(True)`, `wlan.connect(...)`, poll up to retry_timeout seconds (10). Return the IP if connected, else None.

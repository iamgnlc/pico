# Quick 260721-c43: Fix WiFi Connect Timeout and Retry — Summary

## One-liner

`_wifi_connect` now uses a 30 s initial window and performs a single full CYW43 radio-reset retry (disconnect → active(False) → 1 s settle → active(True) → reconnect → 10 s poll) before returning `None`, catching both the tail of cold-boot associations and stuck-state association failures without slowing the already-connected fast path.

## What Changed

**One file modified: `bootstrap.py`**

Function scope: `_wifi_connect(ssid, password, timeout=…)` only. `fetch()`, imports (`network`, `time`, `urequests`), and the entire lower half of the module are byte-identical.

**Behavioral changes:**

1. **Default `timeout` bumped `20` → `30`.** Catches the CYW43 cold-association tail (90th percentile ≈ 30 s per the operator's diagnosis) without slowing the healthy-network happy path (which associates in a few seconds and breaks out of the loop early).

2. **Fast path is now an explicit early return.** The old shape (`if not wlan.isconnected():` then the `return wlan.ifconfig()[0] if wlan.isconnected() else None` tail) still worked, but the new shape returns immediately when already connected — no dead code paths after the mid-session hot spot.

3. **One-shot recovery on initial-poll failure.** After the initial 30 s window fails, the function performs exactly one:
   - `wlan.disconnect()` — clear any queued/stuck CYW43 attempt
   - `wlan.active(False)` — tear the interface down
   - `time.sleep(1)` — CYW43 firmware settle window
   - `wlan.active(True)` — bring the interface back up fresh
   - `wlan.connect(ssid, password)` — re-issue the same credentials
   - 10-second poll loop with 1 s cadence, same shape as the initial loop
   Then returns the IP if connected, else `None`.

**Preserved invariants (all pass Rule 2 correctness check):**

- Signature `_wifi_connect(ssid, password, timeout=30)` — third positional keeps its default. Callers unchanged.
- Return contract: `str` IP on success, `None` on failure. `bootstrap.fetch()` line 33's `if not ip:` fallback still fires exactly when it should.
- Fast path (already-connected on entry): no reset, no extra sleep, no radio churn. Wall-clock cost of a mid-session tick is identical to before.
- `network.WLAN(network.STA_IF)` — station mode unchanged; no AP-mode swap.
- No new imports. No `print`/logging. No docstrings or type hints (MicroPython idiom preserved).
- Worst-case failure-path wall-clock: `30 + 1 + 10 = 41 s`, inside the ≤ 45 s bound the plan defined.

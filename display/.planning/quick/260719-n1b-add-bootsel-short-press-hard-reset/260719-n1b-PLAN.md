<objective>
Add a BOOTSEL short-press hard reset to the running app. Any detected press of the on-board BOOTSEL button reboots the Pico W via `machine.reset()`. The check lives inline at the top of the existing `while True:` scheduler tick in `main.py`, using the same 100 ms `_POLL_MS` cadence — no dedicated timer, no IRQ, no asyncio (respects D-13/14/15).

Purpose: give the operator a physical "escape hatch" reboot without unplugging USB. The BOOTSEL button is unused during normal runtime (it only has hardware meaning at power-on for mass-storage mode), so polling it costs nothing and doesn't fight any existing IRQ or Pin config.

Output: a two-token diff to `main.py` — one added `import rp2`, one extended `from machine import Pin` → `from machine import Pin, reset`, and one new predicate at the top of the tick loop.
</objective>
